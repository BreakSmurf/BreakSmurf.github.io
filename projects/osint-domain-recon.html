OSINT-разведка домена

Инструмент пассивной разведки: собственный WHOIS-клиент по сокетам (без сторонних библиотек), DNS-записи и поиск поддоменов через логи Certificate Transparency (crt.sh). Все источники — публичные, никакого активного взаимодействия с инфраструктурой цели.

Python 3WHOIS-протоколDNScrt.sh

// как это устроено

Логика работы

  • WHOIS: сначала спрашиваем whois.iana.org, какой сервер отвечает за конкретный TLD, затем идём на него по referral за реальной записью — так работает любой WHOIS-клиент.
  • DNS: A/AAAA-записи получаем через стандартный socket.getaddrinfo без внешних зависимостей; MX/NS/TXT — через dnspython, если он установлен.
  • Поддомены: любой публично выпущенный TLS-сертификат попадает в логи Certificate Transparency и содержит доменное имя — поиск по %.domain.com на crt.sh находит поддомены, о которых никогда не публиковалось прямых DNS-записей.
  • Полный отчёт (WHOIS + DNS + список поддоменов) сохраняется в JSON.
